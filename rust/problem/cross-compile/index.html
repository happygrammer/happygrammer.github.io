Rust에서 툴체인 설정을 이용한 크로스 컴파일 방법

2021-01-16

특정 OS에서 개발한 프로그램을 다른 OS 환경에서 실행 하고 싶은 경우가 있다. 이를 위해서는 rustup이라는 명령어를 이용해서 OS 환경에 맞는 툴체인을 설정해 빌드 해야 한다.

rustup 명령어 알아보기

rustup 명령어는 러스트 툴체인 인스톨러 명령어이다.

$ rustup
rustup 1.23.1 (3df2264a9 2020-11-30)
The Rust toolchain installer

USAGE:
    rustup [FLAGS] [+toolchain] <SUBCOMMAND>

FLAGS:
    -v, --verbose    Enable verbose output
    -q, --quiet      Disable progress output
    -h, --help       Prints help information
    -V, --version    Prints version information

ARGS:
    <+toolchain>    release channel (e.g. +stable) or custom toolchain to set override

SUBCOMMANDS:
    show           Show the active and installed toolchains or profiles
    update         Update Rust toolchains and rustup
    check          Check for updates to Rust toolchains
    default        Set the default toolchain
    toolchain      Modify or query the installed toolchains
    target         Modify a toolchain's supported targets
    component      Modify a toolchain's installed components
    override       Modify directory toolchain overrides
    run            Run a command with an environment configured for a given toolchain
    which          Display which binary will be run for a given command
    doc            Open the documentation for the current toolchain
    man            View the man page for a given command
    self           Modify the rustup installation
    set            Alter rustup settings
    completions    Generate tab-completion scripts for your shell
    help           Prints this message or the help of the given subcommand(s)

서브커맨드 옵션중에 override 옵션은 툴체인을 사용자 디렉터리에 적용할 수 있게 하는 명령어이다. 러스트는 릴리즈 채널에 따라 Nightly, Beta, Stable로 나뉜다. 러스트 개발자는 대부분 Stable 채널을 이용한다. 따라서 다음과 같이 stable 채널 기준으로 툴체인을 설정을 할 수 있다.

$ rustup override set stable // 사용자 repo에 stable 채널 툴체인으로 설정(대부분 stable을 사용)
$ rustup override set nightly // 사용자 repo에 nightly 채널 툴체인으로 설정

만약 사용자 환경의 디폴트 툴체인 채널을 변경하고자 한다면 default라는 옵션으로 디폴트 툴체인 채널을 설정할 수 있다.

$ rustup default nightly // nightly 툴체인
$ rustup default stable // stable 툴체인

다음 명령어로 툴체인 채널을 확인할 수 있다.

$ rustup show

component 옵션으로 툴체인 설치 관리를 수행할 수 있다.

SUBCOMMANDS:
    list      List installed and available components
    add       Add a component to a Rust toolchain
    remove    Remove a component from a Rust toolchain
    help      Prints this message or the help of the given subcommand(s)

이중 2번째 인자로 list 옵션을 전달하면 설치 가능한 툴체인 목록을 확인할 수 있다.

$ rustup component list

miri-x86_64-apple-darwin

...

Windows용으로 빌드

다음 명령어는 Window용으로 빌드를 위한 x86_64-pc-windows-gnu 를 설치할 수 있다.

$ rustup target add x86_64-pc-windows-gnu
$ brew install mingw-w64

x86_64-pc-windows-gnu 채널을 설치하고 나서 mingw-w64 설치를 진행한다. 이때 ./cargo/config 설정은 다음과 같다.

[target.x86_64-pc-windows-gnu]
linker = 'x86_64-w64-mingw32-gcc'

빌드와 실행을 해본다.

$ cargo build --target=x86_64-pc-windows-gnu;
# cross 컴파일을 했으으므로 현재 OS에서 실행 불가.
# cargo run --target=x86_64-pc-windows-gnu;

최종적으로 다음 위치에 윈도우 실행 파일인 exe 파일이 생성된다.

./target/x86_64-pc-windows-gnu/debug/<프로그램명>.txt

Linux용으로 빌드

brew를 이용해 의존 파일을 설치한다.

$ brew install FiloSottile/musl-cross/musl-cross  // 필자는 약 15분 소요됨.
$ brew install gcc
$ brew cleanup

rustup 명령어를 이용해 target을 추가한다.

$ rustup target add x86_64-unknown-linux-musl
#rustup target add x86_64-unknown-linux-musl --toolchain=nightly
$ rustup toolchain install stable-x86_64-unknown-linux-musl // 삭제는 install 대신 uninstall 이용

이때 빌드 설정은 다음과 같다. ./cargo/config

[target.x86_64-unknown-linux-musl]
linker = "x86_64-linux-musl-gcc"

그리고 빌드와 실행을 해본다.

$ cargo build --target=x86_64-unknown-linux-musl
# cross 컴파일을 했으으므로 현재 OS에서 실행 불가.
# cargo run --target=x86_64-unknown-linux-musl

최종적으로 다음 위치에 리눅스 실행 파일이 생성된다. 리눅스 용으로 빌드 했으므로 확장자가 존재하지 않는다.

./target/target/x86_64-unknown-linux-musl/debug/<프로그램명>
