[언어 모델] Bag-Of-Words 모델, n-gram 모델

2020-01-31

언어 모델(language model)은 단어 시퀀스(word sequence)를 이용한 확률 모델입니다. 단어 시퀀스는 P(w1,w2,w3,…,wn)와 같이 n개의 단어를 의미하며, n개의 단어를 하나의 시퀀스로 보고 각 시퀀스에 확률 값을 할당합니다. 이러한 시퀀스들의 확률 값들을 언어 모델이라 합니다. 언어 모델은 기계 번역, 음성 인식, 문서 요약, 질의 응답, 오타 교정(spell correction), 손글씨 인식(handwriting recognition) 등과 같이 최종 출력인 단어열을 예측해야 하는 태스크에 사용됩니다.

Bag-Of-Words(BoW)

Bag-Of-Words(BoW) 모델은 이미 분류 또는 정보 검색에서 사용해오던 언어 모델입니다. 정보 검색에서는 문서에 나타난 단어들의 분포들을 고려해 문서 간의 유사도 계산에 사용됩니다. 다음 두 문장은 어순은 다르지만 BoW 관점으로 동일한 의미의 문장입니다.

  • 동해 물과 백두산이 마르고 닳도록
  • 닳도록 마르고 백두산이 물과 동해

위 두 문장은 모두 아래와 같이 동일한 빈도수를 가집니다. 빈도수 요소인 Bag-Of-Words 벡터는 아래와 같습니다.

1 1 1 1 1

Bow 관점에서 세 문서간의 유사도를 비교한다면 두 문서에 사용된 단어(중복 고려 안함) 개수 사이즈의 Bag Of Words 벡터를 만든 후 각 문서에서 얻은 BoW 벡터간의 유사도를 구하면 됩니다. Bag Of Words 벡터를 만들기에 앞서 어휘 벡터를 만들어 보겠습니다.

def getVocabulary(arr):
    arrVocabulary = []
    for obj in arr:
        for key in obj.keys():
            document = obj[key]
            words = document.split()
            for word in words:
                if word not in arrVocabulary:
                    arrVocabulary.append(word)
    return arrVocabulary

전체 어휘 개수가 46개 이므로, 46 길이의 어휘 배열(arrVocabulary)이 만들었습니다. 위 어휘 배열에서 얻은 어휘 사이즈(sizeVocabulary)는 Bag-of-Words 생성시 초기화 벡터의 길이로 사용합니다. 첫번째 문서를 Bag-Of-Words 벡터로 만들면 다음과 같습니다. 다음은 각 문서의 BoW 벡터를 생성해 출력하는 예제입니다.

# txt에 대한 BoW 벡터 생성
def getBagOfWords(documentTxt, arrVocabulary, sizeVocabulary):
    # sizeVocabulary 길이로 Bag-Of-Words 벡터 초기화
    vectorBoW = [0] * sizeVocabulary

    # index 얻기
    words = documentTxt.split()
    for word in words:
        wordIndex = arrVocabulary.index(word)
        vectorBoW[wordIndex] = 1
    return vectorBoW
    

# 각 document에 대한 BoW 벡터 생성 후 전체 문서 벡터 반환
def getBowDocuments(documents, arrVocabulary, sizeVocabulary):
    bowDocuments = []
    # Bag-Of-Words 벡터 생성
    for obj in documents:
        for key in obj.keys():
            documentTxt = obj[key]
            vectorBow = getBagOfWords(documentTxt, arrVocabulary, sizeVocabulary)
            newBowDocument = {key:vectorBow}
            bowDocuments.append(newBowDocument)
    return bowDocuments

bowDocuments = getBowDocuments(documents, arrVocabulary, sizeVocabulary)
print(bowDocuments)

위 예제에서 확인했듯이 BoW는 문서 집합의 어휘 개수 만큼 벡터 사이즈가 커지는 어휘 벡터 사이즈 크기 문제가 있고, 빈도수로 단어 분포가 반영 디지만 단어 순서가 보장디지 않아 맥락 의미 미반영 문제가 있습니다.

  • 어휘 벡터 사이즈 크기
  • 맥락 의미 미반영

어휘 벡터 사이즈 크기 문제는 큰 문서의 단어를 벡터로 변화할 때 단어를 벡터로 변환을 무한하지 않다는 점입니다. 단어를 벡터로 변환할때 메모리 비용이 있고, 어휘 벡터 사이즈가 커질 수 록 계산 비용이 발생합니다. 맥락 의미 미반영 문제는 단어 시퀀스를 고려하지 않아 문맥에 따라 어떤 의미로 사용될지를 예측할 수 없는 문제입니다.

n-gram 모델

언어 모델의 잘 알려진 예로 n-gram 모델이 있습니다. n-gram 모델은 하나의 단어(w) 앞에 나온 n개의 단어열(word sequence)을 고려해 다음에 나올 단어를 예측하는 확률 모델입니다.

P(W)=P(w1,w2,w3,...,wn)

ngram에서 n은 연속된 단어의 개수를 의미하며 n의 개수에 따라 다음과 같이 부릅니다.

  • n=1 : 1-gram(unigram)
  • n=2 : 2-gram(bigram
  • n=3: 3-gram(trigram)
  • n=4 : 4-gram
  • …

애국가 가사를 n-gram 으로 만드는 함수와 이를 bigram(N=2)으로 호출하는 예는 다음과 같습니다.

def getNgram(N,txt):
    txt = txt.split()
    ngrams = [txt[i:i+N] for i in range(len(txt)-N+1)]
    return ngrams

biGrams = getNgram(2, "동해 물과 백두산이 마르고 닳도록 하느님이 보우하사 우리나라 만세")
print(biGrams)
