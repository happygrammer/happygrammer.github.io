[언어 모델] n-gram 모델, Bag-Of-Words 모델

2020-01-31

언어 모델(language model)은 단어 시퀀스(word sequence)를 이용한 확률 모델입니다. 단어 시퀀스는 P(w1,w2,w3,…,wn)와 같이 n개의 단어를 의미하며, n개의 단어를 하나의 시퀀스로 보고 각 시퀀스에 확률 값을 할당합니다. 이러한 시퀀스들의 확률 값들을 언어 모델이라 합니다. 언어 모델은 기계 번역, 음성 인식, 문서 요약, 질의 응답, 오타 교정(spell correction), 손글씨 인식(handwriting recognition) 등과 같이 단어열을 출력해야 하는 태스크에 사용됩니다.

n-gram 모델

언어 모델의 잘 알려진 예로 n-gram 모델이 있습니다. n-gram 모델은 하나의 단어(w) 앞에 나온 n개의 단어열(word sequence)을 고려하는 확률 모델입니다.

P(W)=P(w1,w2,w3,...,wn)

ngram에서 n은 연속된 단어의 개수를 의미하며 n의 개수에 따라 다음과 같이 부릅니다.

  • n=1 이면 unigram
  • n=2 이면 bigram
  • n=3이면 trigram
  • n=4 이면 4-gram
  • …

n-gramjgs 이전 단어들을 고려해 다음에 나올 단어를 예측할 때 사용 가능합니다. 애국가를 bigram으로 분석해 보면 다음과 같습니다.

lyricsOriginal = "하늘 높이 솟는 불 우리들 가슴 고동치게 하네 이제 모두 다 일어나 영원히 함께 살아가야 할 길 나서자 손에 손잡고 벽을 넘어서 우리 사는 세상 더욱 살기 좋도록"

bigramList = lyricsOriginal.split(" ")

res = [(x, i.split()[j + 1]) for i in bigramList  
       for j, x in enumerate(i.split())
           if j < len(i.split()) - 1] 

print ("bigram 결과 : " + str(bigramList))

Bag-Of-Words(BoW)

Bag-Of-Words(BoW) 모델은 언어 모델에서 시퀀스를 고려하지 않는 언어 모델로 일찍히 컴퓨터 비전(computer vision) 분야에 사용되던 모델입니다. BoW는 정보 검색에서 문서간의 유사도 계산에 사용되었습니다. Bag-Of-Words는 시퀀스 고려 없이 각 단어의 빈도수를 고려하는 모델입니다. BoW 관점으로 다음 두 문장를 보겠습니다.

  • Rome was not built in a day
  • day a in built not was Rome

단어 순서를 고려하지 않습니다. 위 문장은 빈도수를 측정해 벡터로 나타내면 동일 벡터로 표현할 수 있습니다.

Rome was not in a Day am are is
1 1 1 1 1 1 0 0 0

Bag-of-Words은 다음과 같은 제한 사항이 있습니다.

  • 맥락 의미 미반영
  • 어휘 벡터 사이즈 제한

맥락 의미 미반영 문제는 단어 시퀀스를 고려하지 않아 문맥에 따라 어떤 의미로 사용될지를 예측할 수 없는 문제입니다. 어휘 벡터 사이즈 제한 문제는 큰 문서의 단어를 벡터로 변화할 때 단어를 벡터로 변환을 무한하지 않다는 점입니다. 단어를 벡터로 변환할때 메모리 비용이 있고, 어휘 벡터 사이즈가 커질 수 록 계산 비용이 발생합니다.
